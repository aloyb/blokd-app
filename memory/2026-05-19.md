### Meridian Screening Error Fix
**Problem:** LLM sering gagal dengan error "no tool call was made" saat screening cycle, padahal semua candidate di-reject oleh indicators.

**Root cause:**
1. `getTopCandidates()` return empty → masih coba call LLM
2. Ada candidate tapi semua difilter oleh indicators → LLM mau respond text "no deploy" → sistem forced tool_call=required → LLM bingung → error
3. SCREENER agent tidak punya tool "skip" → tidak ada cara untuk LLM skip tanpa deploy

**Fixes yang Applied:**

1. **`index.js`** — Early return kalau `getTopCandidates()` return 0 candidates (line 450-461)
   ```javascript
   if (candidates.length === 0) {
     return "No candidates available right now.\n{thresholds}";
   }
   ```

2. **`definitions.js`** — Tambah tool `skip_cycle` untuk SCREENER
   ```javascript
   {
     type: "function",
     function: {
       name: "skip_cycle",
       description: `Skip the current screening or management cycle.
       Use this when no candidate is good enough to deploy after thorough evaluation.
       Call this INSTEAD of returning plain text — never respond with plain text when skipping.`,
       parameters: {
         type: "object",
         properties: {
           reason: { type: "string", description: "Why this cycle is being skipped" }
         },
         required: ["reason"]
       }
     }
   }
   ```

3. **`agent.js`** — Tambah `skip_cycle` ke SCREENER_TOOLS set
   ```javascript
   const SCREENER_TOOLS = new Set([
     "deploy_position", "skip_cycle", "get_active_bin",
     "get_top_candidates", ...
   ]);
   ```

4. **`executor.js`** — Handle `skip_cycle` tool call
   ```javascript
   case "skip_cycle": {
     const reason = args?.reason ?? "No reason provided";
     return { skipped: true, reason };
   }
   ```

5. **`index.js`** — Update screening prompt + onToolFinish handler
   - Prompt: tambah "If no candidate qualifies, you MUST call the skip_cycle tool"
   - `onToolFinish`: set screenReport kalau `skip_cycle` called
